## Key Learnings
- **磁盘**：C盘 defense_ppt.zip（9.8G）是伪装成 zip 的文件夹，实为一次失败打包，内容是当前 AppData 的重复缓存副本（WPS、ForzaHorizon4 游戏缓存等），与正在用的目录逐项对比确认是重复后删除，释放 9.8G。C盘可用 17G→29.8G
- **清理副作用**：清 User Temp 会删掉本会话的 bash 输出临时文件，导致该次命令读不到输出（无害，重扫即可）
- **OG 现实**：微信对分享卡片有缓存，未备案域名朋友圈受限；SPA 全站共用一套 index.html 标签
- **锚点重排（最重要）**：避坑分类用 `pitfall-{分类序}-{条目序}` 定位。删/加分类会让后续所有分类索引位移，`searchIndex.js` 手写别名的锚点必须同步重排，否则搜索跳转错位
- **校验脚本坑**：`searchIndex.js` 里 `import ... from './content'` 无 .js 后缀，Node 原生 ESM 报 ERR_MODULE_NOT_FOUND，只能单独 import content.js 校验锚点，别名映射手动核对

## Solutions & Fixes
- OG：`index.html` 加 og:*/twitter:*/description 标签，构建后 index.html 0.5KB→1.84KB
- 避坑：多轮增删改，最终 content.js 保留 6 分类；searchIndex.js 手写别名全部重写对齐新索引
- 每轮改动都 npm run build + 脚本校验锚点命中，确认无悬空指向再推送

## Key Learnings
### Supabase 列名错误导致 401
- 部署的 MessageWall JS 查询 `id,name,content` 但真实列是 `id,major,text`
- 原因：CDN 缓存了旧版本 JS，列名不匹配
- 解决：CDN 刷新后自动恢复，`select=*` 返回 200

### Supabase REST API 手动设时间戳
- 用 publishable key + fetch() 可直接 POST 设 `created_at`
- 用于种子数据：让留言时间戳错开（7/8~7/11），显自然

### GitHub Pages CDN 缓存
- 推送后 index.html 指向新 JS 文件名，但浏览器可能加载缓存的旧 HTML
- CDN 边缘节点刷新需 1-3 分钟
- 玩法：构建后 `?t=Date.now()` 参数可绕缓存检查新文件

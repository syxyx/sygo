# 会话日志：域名绑定与仓库迁移

**日期**：2026-07-09
**Quick Reference**：sygo.top、域名绑定、仓库迁移、syxyx、GitHub Pages、万网DNS、Vercel、Cloudflare、微信ICP、微信号更新

## Decisions Made
1. **购买域名 sygo.top（万网）并绑定 GitHub Pages**，作为唯一正式域名
2. **创建 GitHub 组织 syxyx**，仓库从 HHX111-CMD/sysc → HHX111-CMD/sygo → syxyx/sygo，缩短 GitHub Pages URL
3. **放弃 Vercel 和 Cloudflare Pages**：试过部署+绑定域名，但都无法解决微信 ICP 备案拦截问题，没必要维护多个平台
4. **微信号从 15364075803 改为 a3399388639**

## Key Learnings
- **微信 ICP 拦截无法绕过**：问题在域名本身（国内注册的 .top 域名），与托管平台无关。Vercel/Cloudflare/GitHub Pages 都会被拦。只有备案能解决。
- **Vercel 默认域名 *.vercel.app 在中国被墙**，自定义域名走 Vercel IP (76.76.21.21) 不受影响
- **Cloudflare Pages 短域名 *.pages.dev 可用**，但同样被微信拦截
- **DNS 传播**：万网改 DNS 记录后需几分钟到几小时全球生效
- **GitHub Pages CDN 有缓存**：改自定义域名后旧 301 跳转可能缓存几分钟，需耐心等待
- **GitHub 组织创建**：类型选 "A business or institution" → 仍有免费方案
- **仓库转移可能卡住**：备选方案是在新组织直接建新仓库 push
- **自定义域名绑定后 vite base 必须是 `/`**：否则 JS/CSS 加载 404

## Solutions & Fixes
- 使用 aliyun CLI 管理万网 DNS 记录（需要 AccessKey）
- GitHub Pages 自定义域名设置：`gh api repos/syxyx/sygo/pages -X PUT -F cname=sygo.top`
- 清除 GitHub Pages CDN 缓存：删除后重建 Pages 配置
- Vercel 默认开启 SSO 保护：`vercel project protection disable --sso`
- 浏览器缓存导致显示旧页面：Ctrl+Shift+Delete 清除缓存或使用无痕模式

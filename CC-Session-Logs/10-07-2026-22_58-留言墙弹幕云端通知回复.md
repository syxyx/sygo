## Quick Resume Context（2-3 句摘要）
给 sygo.top 首页加了「💬 留言墙」并全部上线：访客留言飘成弹幕、下方可展开全部列表；有人留言即通过 Server酱推到站长微信；站长在留言框打暗号「回复」+密码登录后可在网页内公开回复。后端全用 Supabase（永久免费），通知用 Server酱（免费）。三件套均已实测通过。

## Decisions Made
- **后端选 Supabase**：GitHub Pages 纯静态无后端 → 需免费 BaaS。LeanCloud 国际版已关停、腾讯云 CloudBase 免费版有阉割+要实名，最终定 Supabase（永久免费、免绑卡、免实名）。
- **微信通知选 Server酱/方糖**：PushPlus 要实名收费被弃。Server酱 SENDKEY(SCT 开头) + `sctapi.ftqq.com/<KEY>.send`，客户端 GET + `no-cors` 推送。
- **公开回复用「暗号+密码」而非纯暗号**：纯前端暗号只能唤 UI 挡不住 API 篡改；真安全靠 Supabase Auth 登录 + RLS(仅 `auth.uid()` 非空可 update)。暗号「回复」只负责弹密码框。
- **回复显示在列表、不在弹幕**：问答成对更清楚。
- **弹幕懒加载**：SDK 拆独立 chunk，不拖累首屏（主包 gzip ~101KB 不变）。

## Key Learnings
- 纯静态站的第三方通知只能从访客浏览器发 → 只有支持 GET 的服务(Server酱/PushPlus)能配 `no-cors`；企业微信/钉钉 POST webhook 浏览器 CORS 过不了。
- Supabase publishable key(`sb_publishable_...`)是新版 anon key，前端公开安全，靠 RLS 控写。
- 数据层一律 `select('*')`：加 `reply` 字段前后都不报错，避免"列不存在"打断加载。
- RLS 只加了 public read + public insert（不可改删）→ 天然防访客篡改；给站长加 `owner update` 策略(`auth.uid() is not null`)后仅登录站长可写回复。
- Bash 工作目录会跨调用保留：`cd` 进 node_modules 后 `npm run build` 会误跑子包脚本 → 之后命令都显式 `cd` 项目根。

## Solutions & Fixes
- 弹幕引擎：`.danmaku-area`(overflow hidden) + `@keyframes danmakuFloat` translateX 到 `calc(-100vw - 100%)`；JS 定时循环放送 + 发言即飞 + `onAnimationEnd` 回收。
- 密度太密：SPAWN_MS 1600→2600、MAX_ACTIVE 22→14。
- 手机暂停失灵：弃 CSS `:hover`（触屏卡住），改 JS `onMouseEnter/onTouchStart` 控 `animationPlayState`（桌面移入/手机按住暂停）。
- 部署校验用 bundle hash 比对（`curl sygo.top | grep index-*.js`），注意 hash 含下划线要 `[A-Za-z0-9_]`。
